🔐 Authentication

POST /auth/register Register new user Public ▼
Description
Creates a new user account (facility or physician). Validates email uniqueness, password strength, and required field presence. On success, sends a 6-digit OTP to the provided email. User cannot log in until OTP is verified. Also auto-creates a user_preferences row with default settings.
Request Body
Field Type Required Description
email string Required Must be unique, valid email format
password string Required Min 8 chars, must include uppercase, lowercase, number, special char
confirm_password string Required Must match password
user_type enum Required facility | provider
terms_accepted boolean Required Must be true to proceed
Response 201
"user_id": "uuid", "email": "user@example.com", "user_type": "provider", "email_verified": false, "message": "OTP sent to your email. Please verify to continue."
Status Codes
201 Created
400 Validation error (field-level errors)
409 Email already registered
🗄Writes: users · otp_verifications (purpose=registration) · user_preferences (defaults) · user_legal_acknowledgements
POST /auth/verify-otp Verify email OTP Public ▼
Description
Validates the 6-digit OTP for registration or password reset. Tracks attempt count. After 3 failed attempts, the OTP is invalidated. On success for registration, returns access and refresh tokens. On success for password_reset, returns a short-lived reset token only.
Request Body
Field Type Required Description
email string Required Email the OTP was sent to
otp_code string Required 6-digit code
purpose enum Required registration | password_reset
Response 200 (registration)
"access_token": "eyJ...", "refresh_token": "eyJ...", "user_type": "provider", "profile_status": "incomplete", "message": "Email verified successfully."
Status Codes
200 OK
400 Invalid or expired OTP
429 Max attempts reached — OTP invalidated
🗄Updates: users.email_verified=true · otp_verifications.is_used=true · Creates: user_sessions
POST /auth/resend-otp Resend OTP code Public ▼
Description
Invalidates the previous OTP and issues a new 6-digit code. Rate-limited to prevent abuse.
Request Body
Field Type Required Description
email string Required
purpose enum Required registration | password_reset
Status Codes
200 OK — new OTP sent
404 Email not found
429 Rate limit — too many resend requests
🗄Invalidates previous otp_verifications row · Creates new otp_verifications row
POST /auth/login Login — get access & refresh tokens Public ▼
Description
Authenticates user with email and password. Returns JWT access token (short-lived) and refresh token (long-lived). Creates a new user_sessions row. Does not indicate which field is wrong on failure (security).
Request Body
Field Type Required Description
email string Required
password string Required
device_type enum Optional ios | android | web — stored in session

Response 200

Status Codes
200 OK
401 Invalid credentials
403 Email not verified
403 Account suspended
🗄Creates: user_sessions · Updates: users.last_login_at
POST /auth/token/refresh Refresh access token Public ▼
Description
Uses a valid refresh token to issue a new short-lived access token. Validates the token against user_sessions (checks is_revoked, expires_at). Returns 401 if the session has been revoked.
Request Body
Field Type Required Description
refresh_token string Required
Response 200
"access_token": "eyJ..."
Status Codes
200 OK
401 Invalid, expired, or revoked refresh token
POST /auth/logout Sign out — revoke session Auth Required ▼
Description
Revokes the current session by setting user_sessions.is_revoked = TRUE. The access token will remain technically valid until its short TTL expires, but the refresh token is immediately invalidated.
Request Body
Field Type Required Description
refresh_token string Required The session token to revoke
Status Codes
204 No Content
401 Unauthorized
🗄Updates: user_sessions.is_revoked = TRUE

POST /auth/reset-password Set new password after OTP verified Public ▼
Description
Sets a new password. Requires the short-lived reset token returned from /auth/verify-otp (purpose=password_reset). Revokes all existing sessions after successful reset.
Request Body
Field Type Required Description

400 Passwords don't match / weak password
401 Invalid or expired reset token
🗄Updates: users.password_hash · Revokes all user_sessions for this user

✅ Assignments

POST /shifts/{shift_id}/join Join a shift — atomic auto-assignment Auth Required · Provider ▼
Description
The core business action. Runs 4 sequential guards atomically before creating the assignment. No request body needed. On success, shift status flips to filled and assignment is created automatically.
Guards (run in order — all must pass)
🛡G1 — providers.verification_status = 'approved'. Else → 403 PROVIDER_NOT_VERIFIED
🛡G2 — No row in provider_blacklist for (this facility_id, this provider_id). Else → 403 PROVIDER_BLACKLISTED
🛡G3 — No existing shift_assignment for this provider with overlapping date+time range. Else → 409 SHIFT_OVERLAP with conflicting shift details
🛡G4 — shifts.status = 'published' AND no existing assignment row for this shift_id. Else → 409 SHIFT_ALREADY_FILLED
Response 201 (all guards pass)
"assignment_id": "uuid", "shift_id": "uuid", "status": "assigned", "assigned_at": "2026-03-01T14:32:00Z", "message": "Done! You have successfully joined the shift."
Error Responses
// 409 SHIFT_OVERLAP "error": "SHIFT_OVERLAP", "message": "Unable to join. You already have an assigned shift which overlaps.", "conflicting_shift": { "shift_id": "uuid", "shift_date": "Mar 7th, 9:00AM - 5:00PM" }, "requested_shift": { "shift_date": "Mar 7th, 9:00AM - 5:00PM" }
Status Codes
201 Created — assignment made
403 Provider not verified
403 Provider blacklisted
409 Shift overlap conflict
409 Shift already filled
🗄Atomic transaction: INSERT shift_assignments · UPDATE shifts.status=filled

🔐 Authentication

POST/auth/registerRegister new userPublic▼
Description
Creates a new user account (facility or provider). Validates email uniqueness and password strength. Sends a 6-digit OTP. Auto-creates user_preferences row with defaults.
Request Body
FieldTypeRequiredDescription
emailstringRequiredMust be unique, valid email
passwordstringRequiredMin 8 chars, uppercase + lowercase + number + special char
confirm_passwordstringRequiredMust match password
user_typeenumRequiredfacility | provider
terms_acceptedbooleanRequiredMust be true
Response 201
"user_id": "uuid", "email_verified": false, "message": "OTP sent to your email"
Status Codes
201 Created400 Validation error409 Email already registered
🗄Writes: users · otp_verifications (purpose=registration) · user_preferences (defaults) · user_legal_acknowledgements
POST/auth/verify-otpVerify email OTPPublic▼
Description
Validates 6-digit OTP for registration or password reset. Max 3 attempts before invalidation. On success for registration returns access + refresh tokens.
Request Body
FieldTypeRequiredDescription
emailstringRequired
otp_codestringRequired6-digit code
purposeenumRequiredregistration | password_reset
Status Codes
200 OK400 Invalid/expired OTP429 Max attempts — OTP invalidated
🗄Updates: users.email_verified=true · otp_verifications.is_used=true · Creates: user_sessions
POST/auth/resend-otpResend OTP codePublic▼
Description
Invalidates the previous OTP and issues a new 6-digit code. Rate-limited to prevent abuse.
Status Codes
200 OK — new OTP sent404 Email not found429 Rate limit

✅ Assignments

POST/shifts/{shift_id}/joinJoin shift (Provider)Required▼
Description
Provider registers for a shift. Runs all guards atomically in a single transaction: provider verified check, blacklist check, overlap check, and shift availability check.
Guards (in order)
GuardFailure Response
providers.verification_status = 'approved'403 — redirect to profile completion
provider_blacklist check for this facility403 — shift never shown
Overlap check on shift_assignments409 — "You already have a shift at this time"
shifts.status = 'published' + no assignment409 — shift already filled
Status Codes
201 Created403 Not verified / blacklisted409 Filled / time overlap
🗄Writes: shift_assignments (status=assigned) · Updates: shifts.status=filled · Atomic transaction · Notifies facility
